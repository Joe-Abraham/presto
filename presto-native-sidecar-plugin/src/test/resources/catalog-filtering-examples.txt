# Example: Running NativeSidecarPluginQueryRunner with catalog filtering

# To test catalog filtering, run the NativeSidecarPluginQueryRunner with:

# 1. For all functions (default behavior):
# java -cp <classpath> com.facebook.presto.sidecar.NativeSidecarPluginQueryRunner

# 2. For hive catalog only:
# java -Dsidecar.catalog=hive -cp <classpath> com.facebook.presto.sidecar.NativeSidecarPluginQueryRunner

# 3. For presto catalog only:
# java -Dsidecar.catalog=presto -cp <classpath> com.facebook.presto.sidecar.NativeSidecarPluginQueryRunner

# 4. For native catalog only:
# java -Dsidecar.catalog=native -cp <classpath> com.facebook.presto.sidecar.NativeSidecarPluginQueryRunner

# 4. For custom catalog:
# java -Dsidecar.catalog=my_custom_catalog -cp <classpath> com.facebook.presto.sidecar.NativeSidecarPluginQueryRunner

# Once the server is running, you can connect with the CLI and test:
# ./presto-cli --server localhost:8080 --user user

# Test queries to verify catalog filtering:
# SHOW FUNCTIONS;  -- Should show functions from the filtered catalog only
# SELECT * FROM nation LIMIT 5;  -- Basic query should still work
